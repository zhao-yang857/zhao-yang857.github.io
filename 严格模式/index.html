严格模式

2020-12-30
- -

一、严格模式的定义

严格模式（Strict mode）是由ECMA-262规范定义的新兴JavaScript标准，发布于2009年12月第五版。旨在改善错误检查功能并且标识可能不会延续到未来JavaScript版本的脚本。ES5严格模式是限制性更强的JavaScript变体，它与常规JavaScript的语义不同，其分析更为严格。

目前，除了IE6-9，其它浏览器均已支持ES5严格模式

二、严格模式的使用

严格模式 的使用很简单，只有在代码首部加入字符串 “use strict”。有两种应用场景，一种是全局模式，一种是局部模式。

  1. 全局模式
    'use strict'
    2）局部模式
    将”use strict”放到函数内的第一行，如下
    function(){
    'use strict'
}

    三、严格模式的目的

（1）消除Javascript语法的一些不合理、不严谨之处，减少一些怪异行为;
（2）消除代码运行的一些不安全之处，保证代码运行的安全；
（3）提高编译器效率，增加运行速度；
（4）为未来新版本的Javascript做好铺垫。

四、严格模式的变更

（1） 开启严格模式
想开启严格模式，直接在作用域开始的位置写上 字符串 use strict
（2）严格模式的规则
关键字：var 、 with 、this 、参数、八进制、arguments

五、严格模式下的执行机制

  1. 禁止意外创建变量
    在严格模式下，不允许不使用 var 直接声明变量，若只在函数作用域中声明则只有函数作用域才开启严格模式。如下代码在没有使用var就创建了变量：
    'use strict';

str = 'hello world';
console.log(str); //str is not defined
  2. 禁用delete
    delete 用来删除对象中的属性和方法，在非严格模式下对变量进行 delete 不会删除也不会抛出异常。在严格模式下会抛出一个异常。
    如下代码：
    "use strict"

var num = 100;
delete num;
console.log(num) //Delete of an unqualified identifier in strict mode.
  3. 对变量名的限制
    在严格模式下，JavaScript 对变量名的限制变得更加严格，不能使用如下保留字作为变量名：
  • implements
  • interface
  • let
  • package
  • private
  • protected
  • public
  • static
  • yield
    这些保留字在 ES6 中为关键字。
    在非严格模式下编辑器只会给出 warning，开启了严格模式直接报错。
  1. 不可删除对象自带的属性
    "use strict"
delete Object.prototype;
console.log(Object.prototype)
  2. 属性名唯一
    非严格模式下，一个对象可以有若干同名属性和方法，但是都会被最后一个同名属性和方法覆盖，但是在严格模式下，编写期会进行报错提示，
    运行时不会抛出异常。
    如下代码所示：
    "use strict"
var obj = {
    name: '小明',
    name: '小红'
}
console.log(obj.name)
  3. 禁止扩展对象
    之前所学的属性描述符中有提到“防篡改对象”，我们当时将对象设置为了一级保护（不可扩展）。然后我们偏偏对其扩展，发现只是静默失败，
    但是却不会报错；而在严格模式下会抛出一个异常。
    如下代码所示：
    "use strict"

var obj = {}
Object.preventExtensions(obj);
obj.name = '小明';

console.log(obj.name)
  4. 函数中的参数名必须唯一
    在非严格模式下，一个函数中的参数若有同名形参，编辑器只会给出 warning，并且按最后一个相同形参传递的实参值来传入函数体；
    而在严格模式下，若有同名形参，编写期会报错，运行期也会报错。
    如下代码所示：
    "use strict"

function fn1(name, name){
    return name + name;
}

console.log(fn1('hello', 'world'));
  5. arguments 对象值的不同
    “特殊函数” 中我们通过同样的代码，发现了arguments 对象会被内部变量的改变而改变。
    而在严格模式不存在这样的问题，形参传入的是啥 arguments 获取的就是啥，不会被改变。
    如下代码：
    "use strict"

function fn1(value){
    var value = '小红';
    console.log(value); //小红
    console.log(arguments[0]); //小明
}

fn1('小明');
  6. arguments 禁用了 callee 属性
    在递归中我们可以通过 arguments.callee() 方法来实现递归，但是在严格模式下却不能使用该方法。
    如下代码所示：
    "use strict"

function fn1(){
    return arguments.callee;
}

fn1();
    不仅在编写期出现了 error，在运行时也会抛出一个异常。
  7. 函数必须声明在顶层
    我们知道函数声明和函数表达式是两个不同的概念。一般函数声明都在最顶层，ES5前的JS宽松，你可以写在if或for内。当然Firefox的解析方式与其他浏览器不同，见SJ9002。而在严格模式中这些写法将直接报错
    如下代码所示:
    'use strict'
if (true) {
    function func1() { } // 语法错误
}
for (var i = 0; i < 5; i++) {
    function func2() { } // 语法错误
}
  8. this指向了window, 会让this的指向抛空;
    代码如下：
    'use strict'
function foo(){ // 禁用指向了window的this，让this指向undefined
    console.log(this);  //undefined
}
foo()
  9. 严格模式之下禁用with(){}
    代码如下：
    'use strict'
with(Math){
    console.log(random());
    console.log(PI);
}
  10. 禁用八进制
    如下代码所示:
    'use strict'
var num = 022
